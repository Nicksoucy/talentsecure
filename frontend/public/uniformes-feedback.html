1. Vision du module

XGuard gère les uniformes de ~1300 agents. Le système doit :

  1. Traquer chaque pièce physiquement (où elle est, qui la détient).
  2. Décrémenter le stock à la remise, ré-incrémenter au retour après lavage et inspection.
  3. Faire respecter les délais de retour avec escalations automatiques.
  4. Calculer la dette d'un agent et la communiquer à la paie.
  5. Alerter aux moments-clés (RH, Admin, Paie) pour ne rien oublier.

4. Workflow REMISE d'uniforme

Étape par étape, du clic « Nouvelle remise » jusqu'à la signature de l'agent :

4.1. Diagramme de séquence

sequenceDiagram actor Admin actor Agent participant App as TalentSecure participant DB participant GHL Admin->>App: 1. Sélectionne agent + division Admin->>App: 2. Scan codes-barres / cartes par pièce Admin->>App: 3. Définit dueReturnAt (date butoir) Admin->>App: 4. Clic "Finaliser" App->>DB: Crée UniformIssuance + lines App->>DB: applyMovement(OUT) × N → quantityOnHand -= App->>DB: Génère PDF formulaire + upload R2 App->>DB: Crée notif PENDING due-soon (T-24h ouvrables) Admin->>App: 5. Signature employeur (au comptoir) Admin->>App: 6. Clic "Envoyer SMS à l'agent" App->>GHL: SMS lien signature GHL-->>Agent: 📱 SMS avec URL Agent->>App: 7. Ouvre lien, coche consentements, signe App->>DB: signatureStatus=SIGNED

4.2. Actions & notifications

Étape 4.1 : Admin sélectionne agent (autocomplete sur Employee) et choisit division (SECURITE / SIGNALISATION).
Étape 4.2 : Admin scanne le code-barres de chaque pièce OU ajoute via le bouton "+1" sur les cartes du catalogue. Chaque pièce = 1 ligne qty=1.
Étape 4.3 : Admin définit la date butoir dueReturnAt (typiquement 5 jours ouvrables avant fin de contrat OU date fixe selon politique).
Étape 4.4 : Clic "Finaliser" → stock décrémenté, PDF généré, notif due-soon planifiée (status=PENDING, scheduledFor=dueReturnAt - 24h ouvrables).
Notif planifiée à T-24h ouvrables → RH EMAIL CLOCHE : "Rappel — retour d'uniforme demain"
Étape 4.5 : Admin signe au nom de XGuard (employeur) — dialog avec pad de signature au comptoir.
Étape 4.6 : Admin clic "Envoyer SMS" → GHL envoie un lien public à l'agent (token JWT-like, expire 7j).
SMS lien signature → AGENT SMS GHL : "Veuillez signer votre formulaire de prêt : [lien]"
Étape 4.7 : Agent ouvre le lien sur son téléphone, coche les consentements (prélèvement paie, respect uniforme, taille adéquate), signe avec le doigt, soumet.

5. Workflow RETOUR d'uniforme (triage par pièce)

Le retour est l'étape la plus critique : c'est ici que se décide le sort de chaque pièce.

5.1. Diagramme de séquence

sequenceDiagram actor Agent actor Admin participant App as TalentSecure participant DB Agent->>Admin: Apporte ses uniformes au comptoir Admin->>App: 1. Sélectionne agent + remise active App->>App: 2. Génère 1 carte par pièce détenue Note over Admin: 3. Triage carte par carte Admin->>App: Carte 1 (chemise L) → 🟢 Bon Admin->>App: Carte 2 (chemise L) → 🟢 Bon Admin->>App: Carte 3 (chemise L) → 🔴 Endommagé Admin->>App: Carte 4 (ceinture M) → ⚫ Perdu Admin->>App: 4. Clic "Finaliser le retour" App->>DB: Pour Bon : WASH_IN mvmt + UniformWashBatchItem App->>DB: Pour Endommagé : DAMAGED mvmt + dette App->>DB: Pour Perdu : pas de mvmt + dette App->>App: Crée notif wash batch created App->>App: Crée notif retour damaged (admin+RH)

5.2. Triage — règles par condition

ConditionDécisionEffet stockEffet dette agent
🟢 Bon Va dans un lot de lavage automatique WASH_IN — pièce sort de quantityOnHand et entre dans IN_WASHING Aucune
🔴 Endommagé Poubelle directe (déchirée, brûlée, irrécupérable) DAMAGED — sortie définitive Oui : qty × replacementCost
⚫ Perdu Pas physiquement rendue (oubli, vol, perte) Aucun mouvement (déjà sortie à la remise) Oui : qty × replacementCost

5.3. Notifications émises

Si ≥1 pièce 🟢 Bon → ADMIN CLOCHE : "Lot de lavage créé"
Si ≥1 pièce 🔴 Endommagé ou ⚫ Perdu → ADMIN RH EMAIL CLOCHE : "Retour avec items endommagés — dette à confirmer"
SMS signature retour → AGENT SMS GHL

6. Cycle de LAVAGE d'un lot

Une fois qu'un lot existe (créé automatiquement au retour), il suit ce parcours :

stateDiagram-v2 [*] --> CREATED: Auto-créé au finalize d'un retour avec Bon items CREATED --> SENT_TO_LAUNDRY: Admin clic "Envoyer au lavage" + saisit vendor SENT_TO_LAUNDRY --> RETURNED_FROM_LAUNDRY: Admin clic "Marquer revenu" RETURNED_FROM_LAUNDRY --> INSPECTED: Admin inspecte chaque pièce CREATED --> CANCELLED: Admin annule (rare) SENT_TO_LAUNDRY --> CANCELLED: Admin annule INSPECTED --> [*] CANCELLED --> [*]

6.1. Actions par état

État CREATED : Admin peut :
  • Ouvrir le lot et voir les pièces dedans
  • Clic "Envoyer au lavage" + saisir le fournisseur (champ texte libre)
  • Annuler le lot (les pièces re-entrent dans le stock via WASH_OUT_GOOD)
État SENT_TO_LAUNDRY : Admin peut :
  • Clic "Marquer revenu" quand le lot est de retour physiquement
  • Annuler le lot (réinjection stock)
État RETURNED_FROM_LAUNDRY : Admin peut :
  • Clic "Inspecter" → dialog avec 1 carte par pièce
  • Pour chaque pièce, choisir : 🟢 Bon / 🔴 Endommagé (poubelle) / ⚫ Perdu (par le fournisseur)
  • Finaliser l'inspection : les pièces Bonnes ré-intègrent quantityOnHand (WASH_OUT_GOOD) ; les autres sont jetées (WASH_OUT_DAMAGED)

6.2. Notifications cycle lavage

Lot créé → ADMIN CLOCHE
Lot envoyé → ADMIN CLOCHE
Lot revenu (inspection requise) → ADMIN EMAIL CLOCHE
Lot bloqué SENT >7j sans retour → ADMIN EMAIL CLOCHE
Lot RETURNED >3j sans inspection → ADMIN EMAIL CLOCHE

8. Fin d'emploi clôturée

Quand un agent quitte XGuard, on doit clôturer ses remises actives.

sequenceDiagram actor RH participant App participant DB participant Email RH->>App: Clic "Clôturer fin d'emploi" sur la fiche agent App->>DB: Calcule pièces restantes détenues App->>DB: Crée UniformReturn auto avec condition=NOT_RETURNED App->>DB: status=CLOSED_TERMINATION App->>DB: Annule notif due-soon en attente App->>App: Calcule dette finale App->>Email: RH (in-app + email) App->>Email: PAIE (email) Email-->>RH: "Fin d'emploi clôturée — dette $X" Email-->>+Paie: "Prélèvement uniforme — $X"
Bouton "Clôturer fin d'emploi" sur la fiche employé → un seul clic. Toutes les pièces restantes passent en NOT_RETURNED. Calcule la dette finale.
Fin d'emploi clôturée → RH PAIE EMAIL CLOCHE : "Dette uniforme : $X à prélever sur la dernière paie"
